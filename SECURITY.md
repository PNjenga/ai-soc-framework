\# Security Policy



\## Supported Versions



The AI-Driven SOC Framework follows semantic versioning. Only the latest stable release receives active security updates.



| Version | Supported          |

| ------- | ------------------ |

| 1.0.x   | ✅                 |

| 0.2.x   | ❌                 |

| 0.1.x   | ❌                 |

| < 0.1   | ❌                 |



\## Reporting a Vulnerability



If you discover a security vulnerability in the AI-Driven SOC Framework, please report it responsibly so we can address it quickly and minimize risk.



\### How to Report



\- \*\*Preferred method\*\*: Use GitHub's \[Private Vulnerability Reporting](https://github.com/PNjenga/ai-soc-framework/security/advisories/new) (recommended for coordinated disclosure).

\- \*\*Alternative\*\*: Email the project maintainer at security@pnjenga.dev (or the email listed in the repository).



Please provide the following details in your report:

\- Description of the vulnerability and its potential impact

\- Steps to reproduce the issue

\- Affected versions / components (e.g., ingestion pipeline, ML workflows, SOAR automation)

\- Any suggested mitigations or patches



\### What to Expect



\- You will receive an acknowledgment within \*\*48 hours\*\*.

\- We aim to provide an initial assessment and triage within \*\*7 business days\*\*.

\- If the vulnerability is validated, we will develop a fix and coordinate a responsible disclosure timeline.

\- You will be credited in the security advisory and release notes (unless you prefer anonymity).



This project emphasizes verification, validation, and governance practices. All reported issues will be tracked with full traceability through our version control processes.



Thank you for contributing to the security of this AI-Driven SOC Framework!
